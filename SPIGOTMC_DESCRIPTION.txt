[SIZE=6][COLOR=#00BFFF]Overview[/COLOR][/SIZE]

[B]ServerBooster[/B] is a comprehensive server optimization plugin designed to dramatically improve your Minecraft server's performance. This remastered edition has been completely rewritten with modern optimizations and full support for the latest Minecraft versions.

[SIZE=5][COLOR=#32CD32]Why ServerBooster?[/COLOR][/SIZE]

[LIST]
[*][B]Zero Dependencies[/B] - Just drop it in your plugins folder and go!
[*][B]Highly Configurable[/B] - Fine-tune every aspect of optimization
[*][B]Smart Optimization[/B] - Only optimizes what needs to be optimized
[*][B]RAM Efficient[/B] - Automatic chunk unloading keeps memory usage low
[*][B]Multi-Platform[/B] - Works on Spigot, Paper, and Folia
[/LIST]

[SIZE=6][COLOR=#00BFFF]Features[/COLOR][/SIZE]

[SPOILER="Entity Optimization"]
[SIZE=5][COLOR=#FF8C00]Entity Optimization[/COLOR][/SIZE]

[LIST]
[*][B]AI Freeze[/B] - Temporarily disables AI for entities far from players
[*][B]Spawn Limiting[/B] - Controls entity spawns per chunk and radius
[*][B]Age Limiter[/B] - Automatically removes old entities
[*][B]Breeding Control[/B] - Limits breeding to prevent entity explosions
[*][B]Smart Restore[/B] - Instantly restores entity AI when players approach
[/LIST]

Entities are frozen using NMS (Native Minecraft Server) methods, ensuring minimal performance impact while maintaining vanilla behavior when players are nearby.
[/SPOILER]

[SPOILER="Item Stacking & Holograms"]
[SIZE=5][COLOR=#FF8C00]Item Stacking & Holograms[/COLOR][/SIZE]

[LIST]
[*][B]Force Merge[/B] - Stacks items beyond vanilla limits (up to 512+)
[*][B]Unstackable Stacking[/B] - Even swords, armor, and potions can be stacked!
[*][B]Holographic Display[/B] - Shows item name and quantity above stacked items
[*][B]Glowing Items[/B] - Optional glow effect for dropped items
[*][B]Smart Pickup[/B] - Correctly handles inventory distribution
[/LIST]

Reduce thousands of item entities into just a few, dramatically improving TPS on servers with mob farms or heavy item drops.
[/SPOILER]

[SPOILER="Block Limiter (NEW!)"]
[SIZE=5][COLOR=#FF8C00]Block Limiter[/COLOR] [COLOR=#FF0000][NEW!][/COLOR][/SIZE]

[LIST]
[*][B]Radius-Based Detection[/B] - Limits blocks within a configurable radius
[*][B]Predefined Categories[/B] - Groups like "hoppers", "pistons", "furnaces"
[*][B]Custom Materials[/B] - Add ANY block by its Material name
[*][B]Global Limits[/B] - Limit building blocks (stone, wood, etc.)
[*][B]Entity Support[/B] - Limits armor stands, item frames, paintings
[*][B]Non-Destructive[/B] - Existing blocks are never removed
[/LIST]

Prevent lag machines and excessive block placement. Fully customizable per-block limits!

[CODE]
# Example configuration
limits:
  hoppers: 8
  pistons: 8
  observers: 8
  armor-stands: 4
  TNT: 4
  SPAWNER: 2
[/CODE]
[/SPOILER]

[SIZE=6][COLOR=#00BFFF]Commands[/COLOR][/SIZE]

[CODE]
/sb help                    - Show available commands
/sb info                    - Show plugin information
/sb tps                     - Display current server TPS
/sb reload                  - Reload all configurations
/sb count [world]           - Count entities in world
/sb limits                  - Show entity limits
/sb check [player]          - Check entities near player
/sb optimize                - Force entity optimization
/sb blockphysics            - Show physics report
/sb detect spawners [type]  - Detect spawners near players
/sb detect redstone         - Detect redstone mechanisms
[/CODE]

[SIZE=6][COLOR=#00BFFF]Configuration Files[/COLOR][/SIZE]

[CODE]
plugins/ServerBooster/
├── config_entity_limiter.yml    # Entity spawn limits
├── config_optimize_entities.yml # Entity AI optimization
├── config_holo.yml              # Item stacking & holograms
├── chunks_optimizer.yml         # Chunk management
├── chunk_block_limits.yml       # Block placement limits (NEW)
├── config_tps.yml               # TPS-based commands
└── lang/                        # Language files
[/CODE]

[SPOILER="Entity Limiter Configuration"]
[CODE]
# Radius in blocks for entity counting
radius: 56

# Worlds where limiter is active
worlds:
  - world
  - world_nether
  - world_the_end

# Default limits for all entities
defaults:
  radius_max: 10
  chunk_max: 5
  cull: 5

# Custom limits per entity type
limits:
  VILLAGER:
    radius_max: 15
    chunk_max: 8
    cull: -1

  ZOMBIE:
    radius_max: 8
    chunk_max: 4
    cull: 3
[/CODE]
[/SPOILER]

[SPOILER="Block Limiter Configuration"]
[CODE]
enabled: true
radius: 56

worlds:
  - world
  - world_nether
  - world_the_end

deny-message: "&cYou cannot place more {block} in this area! &7(Limit: {limit})"
bypass-permission: "serverbooster.blocklimits.bypass"

# Block limits within radius
limits:
  # Predefined categories
  armor-stands: 4
  item-frames: 4
  hoppers: 8
  pistons: 8
  observers: 8
  chests: 4
  furnaces: 4

  # Custom blocks (use Material names)
  TNT: 4
  SPAWNER: 2
  NOTE_BLOCK: 8

# Global limits for building blocks
global-limits:
  enabled: true
  default-limit: 120
  limits:
    stone-variants: 256
    wood-variants: 256
[/CODE]
[/SPOILER]

[SIZE=6][COLOR=#00BFFF]Frequently Asked Questions[/COLOR][/SIZE]

[SPOILER="FAQ"]
[B]Q: Does this work with Folia?[/B]
Yes! ServerBooster automatically detects Folia and uses region-based scheduling.

[B]Q: Will frozen entities move again?[/B]
Yes! When a player approaches a frozen entity, its AI is instantly restored.

[B]Q: Does item stacking cause duplication?[/B]
No! The stacking system has been thoroughly tested to prevent any duplication.

[B]Q: What happens to existing blocks that exceed limits?[/B]
Nothing! Existing blocks are NEVER removed. Limits only apply to NEW placements.

[B]Q: Can I disable specific modules?[/B]
Yes! Set [ICODE]enabled: false[/ICODE] in each module's config file, then [ICODE]/sb reload[/ICODE].

[B]Q: How do I add a custom block to the limiter?[/B]
Add the Material name directly to the limits section:
[CODE]
limits:
  TNT: 4
  SPAWNER: 2
[/CODE]
[/SPOILER]
